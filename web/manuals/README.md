# GitHub Issues and Contributing

Use this guide when you want to report a bug, request a feature, or contribute code.

Repository: <https://github.com/davidcit646/Workflow>  
Issues: <https://github.com/davidcit646/Workflow/issues>

## How to Submit an Issue (Step by Step)
1. Open the issues page.
2. Search first to avoid duplicate reports.
3. Click `New issue`.
4. Write a clear title:
   - `[Bug] ...`
   - `[Feature] ...`
   - `[Docs] ...`
5. Include:
   - Current app version
   - Platform/OS
   - Steps to reproduce
   - Expected result
   - Actual result
6. Add screenshots or logs if useful.
7. Remove private/sensitive data before posting.
8. Submit and watch for follow-up questions.

## Issue Conventions
- One issue per bug/request.
- Keep titles specific and concise.
- Use numbered reproduction steps.
- Mention impact/severity (blocker, high, medium, low).
- Include regression details when possible.

## Contributing Code
1. Fork the repo and create a branch.
2. Install dependencies: `npm install`.
3. Make your changes.
4. Run lint: `npm run lint`.
5. Test key user flows locally.
6. Open a pull request with:
   - summary of changes
   - testing notes
   - linked issue number

For full contribution guidance, see: <https://github.com/davidcit646/Workflow/blob/main/CONTRIBUTING.md>.
